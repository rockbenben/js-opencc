# js-opencc 自定义保护字典模板（OpenCC 格式）
#
# 每行一条规则：<key><TAB><value>。# 开头为注释，空行忽略。
# 与 OpenCC 上游字典格式一致，可直接抄上游字典的行。
#
# 用法（Node.js）：本文件会被 createConverter 自动加载并作为 protectedDict 应用。
# 直接编辑本文件即可（解开注释或新增规则）：
#
#   import { createConverter } from "js-opencc";
#   const convert = await createConverter({ from: "cn", to: "tw" });
#
# 要绕过自动加载，显式传第二参数（即使是空数组 []）：
#   const convert = await createConverter({ from: "cn", to: "tw" }, []);
#
# 规则匹配：
#   - 多条规则的 key 有重叠时应用「最长优先」
#   - 同名 key 多条规则后写覆盖前写
#   - from === to 合法（锁定原样）
#   - 禁止使用 PUA 字符（U+E000..U+F8FF），它们是内部占位符

# ============================================================
# 1. 覆盖 OpenCC 默认转换（最常见用例）
# ============================================================
# OpenCC 字典里有些多义字按「取第一候选」处理，未必合你预期。
# protectedDict 强制走你自己的版本。

# cn→tw 默认把简体「干」转成「幹」（STCharacters: 干→幹 乾 干）。
# 想保留「干」原样：
# 干	干

# cn→tw 默认把「发」转成「發」（STCharacters: 发→發 髮）。
# 想强制变「髮」（头发的「髮」）：
# 发	髮

# twp 模式默认把「自行車」转成「腳踏車」（TWPhrases: 自行車→腳踏車）。
# 想保留「自行車」原样不被换词：
# 自行車	自行車

# CNTWPhrases.txt 默认把「视频」转成「影片」；想强制走「影像」：
# 视频	影像

# ============================================================
# 2. 锁定专有名词不被反向简化（t2s 方向）
# ============================================================
# 在 tw / twp / hk → cn 方向上，繁体人名、品牌名会被字符级简化
# （例如「周杰倫」中的「倫」会被转成「伦」）。protectedDict 命中
# 后整体跳过内置字典——只在 t2s 方向有实际效果，cn→tw 方向是
# no-op（cn 里本就没这个字符串）。

# 周杰倫	周杰倫

# ============================================================
# 3. 跨义映射（OpenCC 不存在的规则也能强制）
# ============================================================
# protectedDict 不限于简繁对应，任何字符串映射都能注入——
# 适合术语库、品牌译名、英文缩写等场景。

# USA	美利堅
# JavaScript	JS
# OpenAI	開放人工智能
